# Краткий отчет о проекте "audio-store"

В этом документе представлен обзор проделанной работы, архитектуры и ключевых компонентов проекта.

## 1. Обзор проекта

Проект "audio-store" представляет собой микросервисную архитектуру для интернет-магазина аудиокниг. Были реализованы следующие ключевые части:

- **Визуальный интерфейс (Frontend)**: Созданы статические HTML-страницы и CSS-стили для отображения детальной страницы продукта.
- **Слой данных (Persistence Layer)**: Настроен модуль для работы с базой данных SQLite.
- **Микросервис "Каталог"**: Предоставляет API для доступа к списку аудиокниг.
- **Микросервис "Корзина"**: Предоставляет API для расчета стоимости корзины.
- **Инструменты и документация**: Созданы файлы для управления зависимостями, командами, тестами и документацией.

## 2. Архитектурные компоненты

### Фронтенд (`/src`)
- `index.html`: Основная страница, демонстрирующая верстку страницы продукта.
- `styles/style.css`: Стили, соответствующие предоставленным макетам.

### Слой данных (`/database`)
- **`models.py`**: Определены доменные модели `Audiobook`, `Author`, `Category` с использованием SQLAlchemy.
- **`connection.py`**: Централизованный модуль для управления подключением к БД (`audio_store.db`) и предоставления сессий.
- **`init_db.py`**: Скрипт для инициализации базы данных.
- **`seed_db.py` & `examples.py`**: Скрипты для наполнения базы данных тестовыми данными.

### Микросервис "Каталог" (`/services/catalog`)
- **Порт**: `8002` (изначально `8000`)
- **Основной эндпоинт**: `GET /api/v1/audiobooks` и другие.
- **Описание**: Предоставляет полное API для управления каталогом аудиокниг, авторов и категорий.

### Микросервис "Корзина" (`/services/cart`)
- **Порт**: `8003` (изначально `8001`)
- **Основной эндпоинт**: `POST /api/v1/cart/calculate`
- **Описание**: Принимает состав корзины, валидирует его через сервис "Каталог" и возвращает итоговую стоимость.

### Микросервис "Заказы" (`/services/orders`)
- **Порт**: `8004`
- **Основной эндпоинт**: `POST /api/v1/orders`
- **Описание**: Принимает состав корзины, валидирует его через сервис "Корзина", и транзакционно создает заказ в базе данных.

## 3. Тестирование и документация

- **Тесты (`/tests`)**: Создан `test_database.py` для проверки корректности работы моделей и подключения к БД.
- **Документация**:
  - `README.md`: Общее описание проекта.
  - `database/README.md`: Документация по модулю базы данных.
  - `COMMANDS.md`: Список основных команд для работы с проектом.
  - `CHECKLIST.md`: Чек-лист разработки, сгенерированный из технического задания.

## 4. Как запустить

Для запуска проекта следуйте инструкциям в файле `COMMANDS.md`.
