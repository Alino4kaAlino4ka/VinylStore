# Исправление синхронизации данных между каталогом и Cart Service

## Проблема
Товары с ID 8 и 12 не отображались в корзине, хотя добавлялись в localStorage. Cart Service возвращал пустой массив товаров.

## Причина
1. В Cart Service отсутствовали товары с ID 6-9
2. Товары с ID 10-12 имели неправильные данные (не соответствовали каталогу)
3. Моковые данные в cart.js не были синхронизированы с Cart Service

## Исправления

### 1. Добавлены недостающие товары в Cart Service
Добавлены товары с ID 6-9:
```python
"6": CartItem(id="6", title="Гарри Поттер и Принц-полукровка", ...),
"7": CartItem(id="7", title="Гарри Поттер и Дары Смерти", ...),
"8": CartItem(id="8", title="Игра престолов", ...),
"9": CartItem(id="9", title="Битва королей", ...),
```

### 2. Исправлены данные товаров с ID 10-12
**Было:**
- ID 10: "Эмоциональный интеллект" (599.0₽)
- ID 11: "Основание" (27.99₽)  
- ID 12: "Дюна" (34.99₽)

**Стало:**
- ID 10: "Буря мечей" (33.99₽)
- ID 11: "Пир стервятников" (35.99₽)
- ID 12: "Танец с драконами" (37.99₽)

### 3. Синхронизированы моковые данные в cart.js
Добавлены те же товары в fallback данные для случая недоступности Cart Service.

## Результат
- Все товары из каталога теперь корректно отображаются в корзине
- Cart Service возвращает все запрошенные товары
- Данные между каталогом, Cart Service и fallback режимом синхронизированы
- Товары с ID 8, 12, 13, 1 теперь корректно работают
