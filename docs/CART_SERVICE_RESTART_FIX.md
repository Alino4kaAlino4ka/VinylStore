# Исправление проблемы с перезапуском Cart Service

## Проблема
Cart Service не перезапускался с новыми данными, поэтому товары с ID 8, 12, 22 не находились, хотя были добавлены в код.

## Причина
Cart Service был запущен со старой версией кода и не перезапускался автоматически при изменении файлов. Процесс продолжал работать с устаревшими данными.

## Решение
1. **Изменен порт Cart Service** с 8001 на 8002
2. **Обновлена конфигурация** в cart.js для использования нового порта
3. **Запущен новый экземпляр** Cart Service с актуальными данными

## Изменения

### services/cart/main.py
```python
# Было
uvicorn.run(app, host="127.0.0.1", port=8001)

# Стало
uvicorn.run(app, host="127.0.0.1", port=8002)
```

### src/scripts/cart.js
```javascript
// Было
const API_CONFIG = {
    cart: 'http://localhost:8001',
    orders: 'http://localhost:8002'
};

// Стало
const API_CONFIG = {
    cart: 'http://localhost:8002',
    orders: 'http://localhost:8003'
};
```

## Результат
- Cart Service теперь работает с актуальными данными
- Товары с ID 8, 12, 22 корректно находятся и возвращаются
- Все тесты проходят успешно
- Корзина теперь отображает все добавленные товары
