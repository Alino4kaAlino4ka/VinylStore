# Быстрое исправление проблем с сервисами

## Проблема: Сервисы не запускаются из-за ошибок кодировки

### Решение

1. **Остановите все запущенные сервисы** (если есть):
   - Закройте все окна терминала где запущены сервисы
   - Или выполните в PowerShell: `Get-Process python | Stop-Process`

2. **Запустите сервисы заново**:
   ```bash
   python start_services_final.py
   ```

3. **Проверьте логи** в папке `logs/`:
   - Если видите ошибки - скопируйте их
   - Логи должны быть в UTF-8 теперь

## Если сервис recommender не запускается

Проверьте:
1. Наличие файла `config.env` в корне проекта
2. Содержимое `config.env` должно быть:
   ```
   OPENROUTER_API_KEY=sk-or-v1-...
   ```
3. Запуск сервиса напрямую для проверки:
   ```bash
   cd services/recommender
   python main.py
   ```

## Тестирование после запуска

1. Проверьте health endpoints:
   ```bash
   curl http://127.0.0.1:8000/api/v1/products
   curl http://127.0.0.1:8004/health
   ```

2. Или используйте тестовый скрипт:
   ```bash
   python test_description_generation.py
   ```

3. Или откройте в браузере:
   - Catalog: http://127.0.0.1:8000/docs
   - Recommender: http://127.0.0.1:8004/docs
